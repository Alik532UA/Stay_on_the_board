    feat: Refactor MainMenu, implement dev menu features, and fix build errors

    This commit introduces several significant changes and fixes across the application:

    - **MainMenu Refactoring:**
      - Removed top icon section (`main-menu-top-icons`) and subtitle section (`data-testid="main-menu-subtitle"`)
.
      - Streamlined buttons in `id="main-menu-buttons"` to include only "Controls", "Rules", and "Play".
      - Cleaned up unused script variables and functions (`showDevMenu`, `handleDevMenu`, `handleMenuKeyDown`, `is
Dev`) and resolved Svelte block errors.
    - **Developer Menu Enhancements:**
      - Implemented a new top `FlexibleMenu` for development mode only, featuring links to the main menu, a dev me
nu modal, a test mode toggle, and a clear cache button.
      - Displayed the application version number in the Dev Menu modal.
      - Ensured unique `data-testid` attributes for menu slots (`{position}-menu-slot-{i}`) to distinguish between
 top and bottom menus.
    - **Component & Type Improvements:**
      - Updated `FlexibleMenu` to prioritize `item.dataTestId` when passing `dataTestId` to `MenuButton`, allowing
 for custom test IDs.
      - Added `label` and `dataTestId` properties to the `IMenuItem` interface for enhanced flexibility and testin
g.
    - **Reactivity & Styling Fixes:**
      - Resolved an issue where the test mode button's background color wasn't updating by making the `devMenuItem
s` array reactive.
      - Addressed modal height issues for the DevMenu modal by introducing a `customClass` prop in `modalStore` an
d applying specific CSS overrides to remove height restrictions.
    - **Build & Typing Corrections:**
      - Fixed "Module has no default export" errors by correctly importing `MainMenu` as `MainMenuComponent` in re
levant pages.
      - Corrected TypeScript typing for the `keys` variable in test files (`test/buttons/+page.svelte`).
