# План реалізації: Component "FlexibleMenu"

## 1. Мета
Створити універсальний, гнучкий компонент меню, що може розміщуватися знизу або зверху екрану. Компоннент має підтримувати фіксований набір з 5 кнопок з особливою центральною кнопкою ("Floating Action Button"), бути адаптивним та підтримувати згортання/розгортання.

## 2. Вимоги та Конфігурація

### 2.1. Розміщення та Поведінка
*   **Позиції:**
    *   `'bottom'` (За замовчуванням): Прикріплено до низу екрану. Виїжджає вгору. Центральна кнопка виступає вгору.
    *   `'top'`: Прикріплено до верху екрану. Виїжджає вниз. Центральна кнопка виступає вниз.
*   **Анімація:** Плавний "слайд" за межі екрану (Slide-out).
    *   Bottom: Рух вниз (transform translateY).
    *   Top: Рух вгору (transform translateY).
*   **Згортання (Toggle):** Маленька кнопка-"язичок" на внутрішній стороні панелі (зверху для bottom-меню, знизу для top-меню).
*   **Збереження стану:** Стан (відкрито/закрито) зберігається в `localStorage`.

### 2.2. Структура та Дизайн
*   **Кількість кнопок:** 5 (фіксована сітка).
*   **Центральна кнопка (Index 2):**
    *   Кругла, виступає за межі панелі (Floating effect).
    *   Візуально "лежить поверх" інших елементів.
    *   Більш акцентована (розмір/тінь).
*   **Звичайні кнопки:**
    *   Легкий стиль (без масивних рамок), схожий на таби.
    *   Вміст: Іконка (наразі), з архітектурною можливістю додати підпис (Label) знизу.
*   **Адаптивність:**
    *   **Mobile:** Ширина 100% екрану.
    *   **Desktop:** Ширина `fit-content` (по вмісту), центрована або притиснута до краю (залежно від контексту).

### 2.3. Архітектура та API
Компоннент має бути "дурним" (presentational) наскільки це можливо, приймаючи конфігурацію ззовні.

**Props (Вхідні параметри):**
*   `position`: `'top' | 'bottom'` (default: `'bottom'`)
*   `items`: `IMenuItem[]` (масив з 5 елементів)
    *   Якщо елементів менше 5, рендерити пусті плейсхолдери.
    *   Якщо більше 5, брати перші 5 або викидати помилку (dev mode).
*   `persistenceKey`: `string` (опціонально, ключ для `localStorage`. Якщо не передано — стан не зберігається).

**Interface `IMenuItem`:**
```typescript
interface IMenuItem {
    id: string;
    icon: string;       // Назва іконки для SvgIcons
    label?: string;     // Опціональний текст (на майбутнє)
    onClick: () => void;
    isActive?: boolean; // Для підсвітки активного стану
    primary?: boolean;  // Явний прапорець для центральної кнопки (або визначати по індексу 2)
}
```

## 3. Технічна Реалізація

### 3.1. Файлова структура
```text
src/lib/components/ui/FlexibleMenu/
├── FlexibleMenu.svelte      // Головний контейнер та логіка
├── MenuButton.svelte        // Компоннент звичайної кнопки
└── FlexibleMenu.types.ts    // Типи (якщо потрібно винести окремо)
```

### 3.2. Стилізація (CSS)
Використання CSS Grid для розмітки 5 колонок.
```css
.menu-grid {
    display: grid;
    grid-template-columns: repeat(5, 1fr);
    /* ... */
}
```
Для центральної кнопки використовувати абсолютне позиціонування або від'ємні марджини для ефекту "виступання".
Використання CSS Custom Properties (`--menu-bg`, `--menu-height`, etc.) для легкого темізації.
